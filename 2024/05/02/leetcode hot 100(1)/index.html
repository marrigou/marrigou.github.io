105. 从前序与中序遍历序列构造二叉树


  1. 主函数 buildTree：
    • 传入前序遍历数组 preorder 和中序遍历数组 inorder，调用辅助函数 buildTreeHelper 来构建二叉树。
  2. 辅助函数 buildTreeHelper：
    • 参数说明：
      • preorder：前序遍历数组。
      • p_start, p_end：当前递归范围内前序遍历的开始和结束索引。
      • inorder：中序遍历数组。
      • i_start, i_end：当前递归范围内中序遍历的开始和结束索引。
    • 步骤：
      1. 递归终止条件： 当 p_start == p_end 时，表示当前子树为空，直接返回 null。
      2. 确定根节点：
        • 根节点的值是 preorder[p_start]，根据前序遍历特性，根节点总是当前范围的第一个元素。
        • 在中序遍历数组中找到根节点的位置 i_root_index，用于划分左右子树。
      3. 划分左右子树：
        • 左子树节点数： 根据中序遍历数组的索引位置计算左子树的节点数 leftNum，即 i_root_index - i_start。
        • 递归构建左子树：
          • 前序遍历范围：[p_start + 1, p_start + leftNum + 1]。
          • 中序遍历范围：[i_start, i_root_index]。
        • 递归构建右子树：
          • 前序遍历范围：[p_start + leftNum + 1, p_end]。
          • 中序遍历范围：[i_root_index + 1, i_end]。
      4. 返回根节点： 构建完成后返回根节点 root。
  3. 递归的逻辑：
    • 根节点选择： 利用前序遍历的第一个节点作为根节点。
    • 子树划分： 利用中序遍历来划分出左子树和右子树。
    • 递归构造： 先构造左子树，再构造右子树，最终还原整个二叉树结构。

236. 二叉树的最近公共祖先


  1. 基本条件：
    • 在递归查找过程中，如果遇到以下三种情况之一，立即返回 root：
      • root 为空：表示当前节点不存在（叶节点的子节点）。
      • root == p 或 root == q：表示找到了其中一个目标节点 p 或 q。
    • 返回的 root 表示在当前子树中找到了 p 或 q。
  2. 递归查找左右子树：
    • 对 root.left 和 root.right 分别递归调用 lowestCommonAncestor，查找 p 和 q 在左右子树中的位置。
    • left 和 right 分别存储了左子树和右子树中查找到的最近公共祖先或目标节点。
  3. 合并结果：
    • 只有右子树有结果： 如果 left 为空，说明 p 和 q 都在右子树，直接返回 right。
    • 只有左子树有结果： 如果 right 为空，说明 p 和 q 都在左子树，直接返回 left。
    • 左右子树都有结果： 如果 left 和 right 都不为空，说明 p 和 q 分别在 root 的左右子树，当前节点 root 就是最近公共祖先。

124. 二叉树中的最大路径和


  1. 递归函数 dfs：

    • 终止条件： 如果当前节点为空，返回 0，因为空节点没有贡献。
    • 计算贡献：
      • 计算左孩子的贡献 leftMax，如果左孩子返回的路径和小于 0，则取 0，表示左边不纳入路径。
      • 计算右孩子的贡献 rightMax，同理，如果右孩子返回的路径和小于 0，则取 0，表示右边不纳入路径。
    • 更新全局最大路径和 res：
      • 计算当前路径和 root.val + leftMax + rightMax，即包含当前节点值、左孩子贡献、右孩子贡献的路径和。
      • 更新 res，使其等于 res 和 root.val + leftMax + rightMax 中的较大值。
    • 返回当前节点的贡献：
      • 返回 root.val + Math.max(leftMax, rightMax)，即当前节点值加上左、右孩子贡献中的较大值，表示在该节点路径向上递归时选择较大的贡献。

  2. 核心思想：

    • 最大路径和的计算： 对于每一个节点，计算其左孩子和右孩子对路径的最大贡献，若某一边的贡献为负数，则不纳入路径。
    • 全局最大值 res 的更新： 对于每一个节点，检查以该节点为路径最高点（包括左右孩子）的路径和，并更新全局的 res。
    • 递归返回值： 每次递归返回的是当前节点能向上提供的最大路径和（包含当前节点）。

239. 滑动窗口最大值


该代码通过一个双向队列来维护滑动窗口中的最大值。双向队列中存储的是数组的索引，通过确保队列中索引对应的值是从大到小排列的，可以在 O(1) 的时间内获取当前窗口的最大值。
滑动窗口的遍历过程中，当遍历到一个新的元素时，首先会维护 queue 中的元素顺序（保证从大到小），然后将当前元素的索引加入 queue。接着检查队首索引是否超出当前窗口范围，超出则移除队首元素。最后，当窗口长度达到 k 时，将当前窗口的最大值添加到结果数组。

这种双向队列方式使得时间复杂度保持在 O(n)，因为每个元素只会被入队和出队一次。

  1. 输入数组的特殊情况处理：
    • 如果输入数组 nums 为 null 或者数组的长度小于 2，则直接返回原数组，因为滑动窗口没有必要进行操作。
  2. 使用双向队列维护最大值：
    • 创建一个双向队列 queue 来存储窗口中元素的索引，以确保队列中的数值（通过索引查找 nums）是从大到小排列的。
    • 这样可以在 O(1) 时间复杂度内获得当前窗口中的最大值。
  3. 遍历 nums 数组：
    • 对于每个元素 nums[i]，首先检查 queue 的尾部元素：
      • 如果 nums[i] 大于等于 queue 尾部元素所对应的数值，说明 queue 中尾部元素不可能是最大值，所以将其移除。
    • 然后将当前元素的索引 i 添加到 queue 的尾部。
  4. 维护窗口范围：
    • 检查队首的元素是否已经不在当前窗口范围 [i - k + 1, i] 内：
      • 如果队首元素的索引小于 i - k，则说明该索引对应的数值已经移出窗口范围，需要从 queue 中移除。
  5. 记录窗口中的最大值：
    • 当窗口大小达到 k 时（i + 1 >= k），将当前窗口的最大值添加到结果数组 result 中。
      • 由于 queue 中的元素从大到小排列，queue 的队首就是窗口中的最大值。
  6. 返回结果数组：
    • 遍历完所有元素后，结果数组 result 中存储的就是每个窗口中的最大值，返回该数组。

  2. **遍历字符串 s**：
    • 使用两个指针 left 和 right 维护滑动窗口的左右边界，初始时 left 和 right 都为 0。
    • right 不断向右扩展，表示滑动窗口的右边界，同时更新 hs 中对应字符的出现次数。
  3. 字符匹配逻辑：
    • 当 s[right] 是 t 中的字符（存在于 ht 中），并且 hs 中该字符的次数未超过 ht 的要求时，增加匹配计数 cnt。
    • cnt 代表了当前窗口中已经匹配到的 t 中字符的个数。
  4. 收缩窗口：
    • 当窗口的左边界字符 s[left] 不在 t 中，或者在 hs 中的次数超过了 ht 中的要求，就可以右移 left 来收缩窗口，直到满足窗口内 t 中字符的要求。

    • 每次当 cnt 等于 t 的长度（说明窗口中已经包含了所有 t 中的字符），就检查当前窗口的长度 right - left + 1 是否小于当前的最小长度 len，如果是，更新最小长度 len 和结果子串 ans。
  6. 返回结果：
    • 遍历结束后，返回 ans，即包含所有 t 中字符的最小窗口子串。

    • i 初始化为数组的起始位置（左边界）。
    • j 初始化为数组的末尾位置（右边界）。
    • res 初始化为 0，用于存储计算出的最大容积。
  2. 使用双指针遍历数组：
    • 在 while (i < j) 的循环中，双指针 i 和 j 会不断向中间靠拢，直到相遇。
    • 每次循环中，计算由 i 和 j 指向的线段所能形成的容器的面积，并更新 res。
  3. 计算面积并移动指针：
    • 计算面积的公式为 (j - i) * min(height[i], height[j])，其中 j - i 是宽度，min(height[i], height[j]) 是高度。
    • 如果 height[i] < height[j]，则移动左指针 i 向右，因为移动较矮的指针可能找到更高的线段，从而可能增加容积。
    • 如果 height[j] <= height[i]，则移动右指针 j 向左，原因同上。
    • 移动较矮的指针是因为容器的高度取决于较短的那根线段，要想找到更大的容积，需要试图找到更高的线段。
  4. 返回最大容积：
    • 当 i 和 j 相遇时，循环结束，res 存储了整个过程中找到的最大容积。

394. 字符串解码


该代码的目的是解码一个格式化的字符串 s，其中包含数字和括号，表示某些子字符串需要重复多次。解码规则如下：k[encoded_string]，其中 k 是重复次数，encoded_string 是需要重复的字符串。

  1. 栈的使用：
    • 该算法采用 双栈 来处理多层嵌套的情况：
      • **stack_multi**：保存每一层括号对应的重复次数 k。
      • **stack_res**：保存每一层括号之前已经构建的字符串，以便在括号结束时进行拼接。
  2. 遍历字符串：
    • 遍历 s 中的每个字符，按以下情况处理：
      • **左括号 [**：表明开始一个新的子字符串：
        • 将当前的 multi（重复次数）压入 stack_multi，并重置 multi 为 0。
        • 将当前的 res（构建中的字符串）压入 stack_res，并重置 res，准备处理新的子字符串。
      • **右括号 ]**：表明一个子字符串构建结束：
        • 弹出 stack_multi 栈顶的重复次数 cur_multi。
        • 将 res 重复 cur_multi 次，并存入 tmp 中。
        • 弹出 stack_res 栈顶的字符串，与 tmp 拼接，更新 res 为拼接后的结果。
      • 数字字符：用于计算当前子字符串的重复次数 multi，考虑到可能有多位数字，需要累积计算。
      • 字母字符：直接加入 res，作为当前子字符串的一部分。
  3. 拼接结果：
    • 在遍历结束后，res 中保存了完整的解码字符串，直接返回。

假设 s = "3[a2[c]]"：

  • 初始：multi = 0，res = ""，stack_multi = []，stack_res = []。
  • 遍历：
    • 3：multi = 3。
    • [：stack_multi = [3]，stack_res = [""]，multi = 0，res = ""。
    • a：res = "a"。
    • 2：multi = 2。
    • [：stack_multi = [3, 2]，stack_res = ["", "a"]，multi = 0，res = ""。
    • c：res = "c"。
    • ]：cur_multi = 2，tmp = "cc"，res = "a" + "cc" = "acc"。
    • ]：cur_multi = 3，tmp = "accaccacc"，res = "" + "accaccacc" = "accaccacc"。
  • 返回结果："accaccacc"。

84. 柱状图中最大的矩形


  1. 栈的设计：
    使用栈来保存柱形的索引，保证栈中的柱形高度是 单调递增 的。这样在遇到一个较低的柱形时，就可以确定栈中存储的柱形不能再向右延伸，必须结算它们的面积。
  2. 遍历数组：
    遍历 heights 数组，对于每个柱形，判断它与栈顶柱形的高度关系：
    • 如果当前柱形高度小于栈顶柱形高度，说明栈顶柱形不能再向右延伸了，需要结算栈顶柱形的面积。
    • 结算方式：弹出栈顶柱形，计算以该柱形为高度的最大矩形面积，然后继续处理栈中下一个柱形，直到栈顶柱形高度不再大于当前柱形高度为止。
  3. 宽度的计算：
    • 当结算一个柱形的面积时，宽度是从当前索引向左延伸到栈中下一个柱形（如果有）之间的宽度。
    • 如果栈为空，说明当前柱形是最矮的，可以延伸到索引 0。
    • 如果栈不为空，宽度为从栈顶柱形的索引下一位到当前索引的距离。
  4. 遍历完数组后的处理：
    当遍历完 heights 数组后，栈中可能还剩一些未结算的柱形。此时，需要继续从栈中弹出柱形并计算它们的面积。
  5. 时间复杂度：
    该算法的时间复杂度为 **O(n)**，因为每个柱形最多只会被压入和弹出栈一次。
  6. 最终结果：
    返回最大的矩形面积 res，即在直方图中可以形成的最大矩形面积。

该代码实现了快速排序（Quick Sort）算法，对给定的整数数组 nums 进行排序。

  • 递归终止条件： start >= end 表示当前子数组的范围已经不需要再进行排序（只有一个元素或无元素），直接返回。
  • 初始化指针： left 指向 start，right 指向 end，并选择 left 指向的值 nums[left] 作为基准元素 temp。
  • 分区过程：
    • 从右向左遍历：移动 right 指针，寻找第一个小于 temp 的元素，并将其移动到 left 指针的位置。
    • 从左向右遍历：移动 left 指针，寻找第一个大于 temp 的元素，并将其移动到 right 指针的位置。
    • 上述两个过程交替进行，直到 left 和 right 相遇。
  • 基准元素归位： 当 left 和 right 相遇后，将基准元素 temp 放到 left 指针位置，此时 left 位置的元素是正确排序的。
  • 递归调用： 对基准元素左右两侧的子数组分别进行递归排序，直至整个数组排序完成。

31. 下一个排列


  1. 寻找转折点：k 指针从数组末尾向前扫描，寻找第一个 nums[k-1] < nums[k] 的位置 k - 1，这是数组中由右向左第一个不符合升序的点，标记为转折点。该位置的元素需要被替换，以找到下一个排列。
  2. 特殊情况处理：当 k == 0 时，表示数组为完全降序排列，这是字典序中的最大排列，因此需要将整个数组逆序，变为字典序中的最小排列（完全升序）。
  3. 交换元素：当找到转折点 k - 1 后，再从 k 向右查找，寻找第一个比 nums[k - 1] 大的元素 nums[u]，然后交换这两个元素，以确保新的排列比当前排列大，同时尽可能小。
  4. 反转尾部元素：在交换后，将 k 位置及其后面的元素进行逆序排列，使得新的排列是字典序中最小的“较大”排列。这样确保得到的结果是紧接当前排列的下一个字典序排列。
  5. **辅助方法 reverse 和 swap**：reverse 方法用于反转指定范围内的元素顺序，swap 方法用于交换两个索引的元素。 我们用一个例子来具体讲解一下代码的执行过程。假设数组 nums 为 [1, 2, 3, 6, 5, 4]，我们想要找出它的下一个字典序排列。
    步骤 1：寻找最右侧非递增序列的拐点
    从后向前扫描数组，寻找第一个位置 k-1，使得 nums[k-1] < nums[k]：
  • 从最后一个元素 4 开始，依次比较 nums[5] (4) 与 nums[4] (5)，nums[4] > nums[5]，继续。
  • 比较 nums[4] (5) 与 nums[3] (6)，nums[3] > nums[4]，继续。
  • 比较 nums[3] (6) 与 nums[2] (3)，nums[2] < nums[3]，找到了位置 k-1 = 2，也就是 nums[2] = 3。
    此时我们找到了拐点位置 k-1 = 2，因为 nums[2] (3) 小于 nums[3] (6)，k = 3。
    步骤 2：寻找合适的交换位置
    在 k 之后的部分 [6, 5, 4] 中，寻找比 nums[k-1] (3) 大的最小的数：
  • 我们找到 4，因为它是 [6, 5, 4] 中比 3 大的最小数，位置为 u = 5。
    步骤 3：交换 nums[k-1] 和 nums[u]
    交换 nums[2] (3) 和 nums[5] (4)，交换后的数组变为 [1, 2, 4, 6, 5, 3]。
    步骤 4：逆序排列后缀
    从 k 开始（位置 3），对数组的后缀 [6, 5, 3] 进行逆序：
  • 6 和 3 交换，得到 [1, 2, 4, 3, 5, 6]。
    最终结果
    得到的数组是 [1, 2, 4, 3, 5, 6]，这是 [1, 2, 3, 6, 5, 4] 的下一个字典序排列。

分步详解
初始数组： [1, 2, 3, 6, 5, 4]

  1. 寻找拐点： k-1 = 2，即 nums[2] = 3 是第一个使 nums[k-1] < nums[k] 的位置。
  2. 寻找要交换的元素： 在 k 之后寻找比 3 大的最小数，找到 4（u = 5）。
  3. 交换： 3 和 4 交换，数组变成 [1, 2, 4, 6, 5, 3]。
  4. 逆序后缀： 从 k = 3 开始，对 [6, 5, 3] 逆序，得到最终结果 [1, 2, 4, 3, 5, 6]。
